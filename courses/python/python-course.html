Python Fundamentals
Installing Python · Variables · Data Types · print()
⚡ Why This Matters Every program you will ever write in Python starts here. Variables store information. Data types tell Python what kind of information it is. And print() lets you see what your program is doing. These three concepts are the foundation of everything — skip them and nothing else makes sense.
Concept Explanation

What Is Python?

Python is a beginner-friendly programming language that reads almost like plain English. It is used everywhere — from websites and mobile apps to artificial intelligence and data analysis. Big companies like Google, Netflix, and Instagram use Python in their backend systems.

Variables — Your Program's Memory

A variable is like a labelled box that holds a value. You give the box a name, and Python remembers what's inside it for you. You can change what's in the box at any time.

python # Creating variables – just write: name = value name = "Denis" # a String (text) age = 25 # an Integer (whole number) height = 1.75 # a Float (decimal number) is_enrolled = True # a Boolean (True or False only) # Python figures out the type automatically – no need to declare it print(type(name)) # <class 'str'> print(type(age)) # <class 'int'>

The Four Core Data Types

  • String (str) — Text surrounded by quotes. Example: "Hello", "Narok"
  • Integer (int) — Whole numbers, no decimals. Example: 42, -7
  • Float (float) — Numbers with a decimal point. Example: 3.14, 99.9
  • Boolean (bool) — Only two possible values: True or False

Printing Output — Talking to the User

The print() function displays information on the screen. It's the most important tool for debugging and showing results. Notice the three ways to combine variables with text:

python – three ways to print variables name = "Denis" age = 25 city = "Narok" # Method 1: Comma-separated (adds a space automatically) print("Hello, my name is", name, "and I am", age) # Method 2: String concatenation (joining with +) print("I live in " + city + ".") # Method 3: f-string (most modern, most readable – USE THIS) print(f"Hello! I am {name}, {age} years old, from {city}.")
🌍 Real-World Example When you open the Safaricom app and it says "Welcome back, Denis!" — that is a variable. The app stores your name in a variable when you register, and then retrieves it every time you log in to personalise your experience.
⚡ Quick Challenge — Try It Now

  • Print a single f-string sentence that uses all four variables
  • Use type() to print the type of each variable
🚫 Common Mistakes to Avoid
  • Forgetting quotes around strings: name = Denis causes an error. Always use: name = "Denis"
  • Mixing types in + concatenation: "Age: " + 25 crashes. Use f-strings instead: f"Age: {25}"
  • Capitalising booleans wrong: Python requires True and False — not true or TRUE
  • Spaces in variable names: my name = "Denis" is invalid. Use underscores: my_name = "Denis"

User Input
input() · Strings · Type Conversion
⚡ Why This Matters A program that can't talk to users is useless. Every app — from M-Pesa to Instagram — needs to take input from real people. This module teaches you how to make your programs interactive, so users can type their own data and get personalised responses.
Concept Explanation

The input() Function

The input() function pauses your program and waits for the user to type something. Whatever the user types gets stored in a variable. The text inside the parentheses is the prompt — what the user sees before they type.

python # Basic input – always returns a STRING name = input("What is your name? ") print(f"Welcome, {name}!")

⚠️ Important: input() Always Returns a String

This is the most important rule in this module. Even if the user types a number, Python treats it as text. You must convert it before doing any maths.

python – type conversion # WRONG – this will crash when you try to add 1 age = input("How old are you? ") print(age + 1) # ❌ TypeError: can't add str + int # CORRECT – convert to integer first age = int(input("How old are you? ")) print(f"Next year you will be {age + 1}.") # ✅
# Convert to float for decimal numbers price = float(input("Enter price in KSh: ")) print(f"With 16% VAT: KSh {price * 1.16:.2f}")

String Manipulation

Strings come with built-in tools (called methods) that let you transform them. These are essential for cleaning user input before using it.

python – useful string methods name = input("Enter your name: ") # user types: " denis " print(name.strip()) # "denis" → removes spaces print(name.title()) # "Denis" → capitalises first letter print(name.upper()) # "DENIS" → all caps print(name.lower()) # "denis" → all lowercase print(len(name.strip())) # 5 → number of characters # Chain methods together clean_name = name.strip().title() print(f"Hello, {clean_name}!")
🌍 Real-World Example When you register on a website and type your email address, the system runs .strip().lower() on it behind the scenes. This makes sure "Denis@Gmail.com" and "denis@gmail.com" are treated as the same email — preventing duplicate accounts.

🚫 Common Mistakes to Avoid
  • Forgetting to convert input: Always wrap numeric input with int() or float()
  • Converting text with int(): int("hello") crashes — only convert when you're sure the user typed a number
  • Forgetting the space in the prompt: Write "Enter name: " (with a space before the closing quote) so the cursor appears after the colon

Control Flow
if/elif/else · Comparison Operators · Boolean Logic
⚡ Why This Matters Right now your programs run top to bottom, doing exactly the same thing every time. Control flow gives your program a brain — the ability to make decisions. This is what separates a script from a real application. Every login system, every quiz, every M-Pesa transaction check uses control flow.
Concept Explanation

if / elif / else — Teaching Python to Decide

Think of if like a fork in the road. Python checks a condition — if it's true, it takes one path; if not, it can check another condition (elif) or fall back to a default (else).

python – structure of an if statement # IMPORTANT: Indentation (4 spaces) tells Python what's inside the if block score = int(input("Enter your exam score: ")) if score >= 80: print("Distinction 🏆") elif score >= 60: print("Credit ✓") elif score >= 40: print("Pass") else: print("Below pass – keep going!")

Comparison Operators

These are the tools Python uses to compare values and produce a True or False result:

  • == — Equal to (note: two equals signs, not one)
  • != — Not equal to
  • > / < — Greater than / Less than
  • >= / <= — Greater than or equal / Less than or equal

Boolean Logic — Combining Conditions

Use and, or, and not to combine multiple conditions into one check:

python – boolean logic in action age = int(input("Enter age: ")) has_id = input("Do you have ID? (yes/no): ").lower() == "yes" # AND – both conditions must be True if age >= 18 and has_id: print("Access granted ✅") elif age >= 18 and not has_id: print("You are old enough but please bring your ID.") else: print("Sorry, you must be 18+ to enter.") # OR – at least one condition must be True day = input("Enter today: ") if day == "Saturday" or day == "Sunday": print("It's the weekend! 🎉")
🌍 Real-World Example When you try to withdraw money from an ATM, the bank runs control flow: if your PIN is correct and your balance is sufficient and your daily limit hasn't been reached, then approve the withdrawal — else show an error. That's nested boolean logic with and conditions, exactly what you just learned.
⚡ Quick Challenge — Try It Now

  • Ask for two numbers and print which one is larger (or if they're equal)
  • Build a simple login: ask for a password, and print "Access granted" only if it matches "vtx2025"
🚫 Common Mistakes to Avoid
  • Using = instead of ==: if name = "Denis" is assignment, not comparison. Always use ==
  • Wrong indentation: Python uses 4 spaces (or 1 tab) for indentation. Mixing them causes errors
  • Forgetting the colon: Every if, elif, and else line must end with :

04
Loops
for loops · while loops · Iteration
⚡ Why This Matters Without loops, if you wanted to print 1,000 numbers, you'd need 1,000 lines of code. Loops let you repeat actions automatically. They are the engine behind sending bulk SMS alerts, processing rows in a spreadsheet, displaying product listings on a website — anything that involves repeating a task.
Concept Explanation

The for Loop — When You Know How Many Times

A for loop repeats for each item in a sequence. Use it when you know exactly how many repetitions you need.

python – for loop with range() # range(start, stop, step) – stop is NOT included for i in range(1, 6): print(f"Counting: {i}") # prints 1, 2, 3, 4, 5 # Even numbers only (step = 2) for n in range(2, 21, 2): print(n) # 2, 4, 6, ... 20 # Loop through a list of items students = ["Denis", "Amina", "Brian", "Fatuma"] for student in students: print(f"Welcome to class, {student}! 👋")

The while Loop — When You Don't Know How Many Times

A while loop keeps running as long as a condition is True. It's perfect for things like "keep asking until the user gives a valid answer."

python – while loop # Count down from 5 count = 5 while count > 0: print(f"Launching in {count}...") count -= 1 # CRITICAL: always update your counter or you get an infinite loop print("🚀 Lift off!") # Keep asking until the user gives a valid answer answer = "" while answer != "yes" and answer != "no": answer = input("Are you ready? (yes/no): ").lower() print(f"You said: {answer}")

break and continue — Controlling Loop Flow

python – break and continue # break – exit the loop immediately for i in range(1, 10): if i == 5: break # stops at 5 – prints 1, 2, 3, 4 print(i) # continue – skip this iteration, go to next for i in range(1, 10): if i % 2 == 0: continue # skip even numbers print(i) # prints 1, 3, 5, 7, 9
🌍 Real-World Example When an e-commerce site like Jumia loads its product page, it loops through a database of hundreds of products and creates an HTML card for each one. That's a for loop in action — iterating over a list and performing an action (creating a card) for each item.
⚡ Quick Challenge — Try It Now
  • Print a multiplication table for 7 (7×1 through 7×10)
  • Write a while loop that keeps asking for a number until the user types 0, then print the sum of all numbers entered
  • Print all numbers from 1 to 50 that are divisible by both 3 and 5
🚫 Common Mistakes to Avoid
  • Infinite while loops: Always make sure the condition eventually becomes False. Always update your variable inside the loop
  • Off-by-one errors with range(): range(1, 10) gives 1–9, not 1–10. The stop value is excluded
  • Modifying a list while looping over it: This causes unpredictable behaviour. Loop over a copy instead

05
Functions
def · Parameters · Return Values
⚡ Why This Matters Without functions, every time you need the same block of code you'd copy and paste it. Functions let you write code once and reuse it everywhere. Professional developers think in functions — every feature in a real app is a collection of small, focused functions working together.
Concept Explanation

Defining and Calling Functions

Use the def keyword to define a function. Give it a clear, descriptive name. Call it by writing the name followed by parentheses. Nothing inside the function runs until you call it.

python – anatomy of a function # DEFINE: create the function (runs nothing yet) def greet_user(name): message = f"Hello, {name}! Welcome to VersaTech Campus." return message # CALL: execute the function with an argument result = greet_user("Denis") print(result) # Call it again with different input – no rewriting needed! print(greet_user("Amina")) print(greet_user("Brian"))

Parameters, Arguments, and Default Values

python – parameters and defaults # Default parameter – used when caller doesn't provide a value def greet(name, language="English"): if language == "Swahili": return f"Habari, {name}!" elif language == "Maa": return f"Sopa, {name}!" return f"Hello, {name}!" print(greet("Denis")) # uses default → "Hello, Denis!" print(greet("Amina", "Swahili")) # → "Habari, Amina!" # Multiple return values def rectangle_stats(w, h): area = w * h perimeter = 2 * (w + h) return area, perimeter a, p = rectangle_stats(10, 5) print(f"Area: {a} | Perimeter: {p}")

Variable Scope — Local vs Global

python – understanding scope company = "VersaTech" # Global – visible everywhere def show_info(): department = "Engineering" # Local – only exists inside this function print(f"{company} – {department}") # can see global company show_info() # print(department) ← ❌ ERROR: department doesn't exist outside the function
🌍 Real-World Example Every time you send an M-Pesa payment, a function runs behind the scenes: it takes your phone number, recipient number, and amount as parameters, checks your balance, deducts the fee, sends the money, and returns a confirmation message. One function, called millions of times a day with different inputs.

🚫 Common Mistakes to Avoid
  • Forgetting to return: A function without return returns None. If you need a value back, always use return
  • Calling before defining: In Python, you must define a function before calling it (unless you're inside another function)
  • Trying to use a local variable outside the function: Variables created inside a function only exist there

Lists & Dictionaries
Lists · Dicts · Accessing & Modifying Data
⚡ Why This Matters Variables hold one piece of data. But what if you need to store 1,000 student names, or an entire user profile? Lists and dictionaries are Python's primary data structures — they are the backbone of virtually every real application. Without them, you cannot build anything meaningful.
Concept Explanation

Lists — Ordered Collections

A list stores multiple values in a single variable, in a specific order. Each item has an index starting from 0 (not 1). Think of it like a numbered queue.

python – working with lists # Create a list students = ["Denis", "Amina", "Brian", "Fatuma"] # Access by index (starts at 0) print(students[0]) # "Denis" print(students[-1]) # "Fatuma" (last item) # Modify the list students.append("James") # add to end students.insert(1, "Eve") # insert at position 1 students.remove("Brian") # remove by value # Useful list operations print(len(students)) # number of items print(sorted(students)) # alphabetical order print("Denis" in students) # True/False – check membership # List comprehension – elegant one-liner scores = [45, 82, 31, 90, 67] passed = [s for s in scores if s >= 50] print(passed) # [82, 90, 67]

Dictionaries — Labelled Data

A dictionary stores data as key-value pairs — like a contact book where you look someone up by name (the key) to get their phone number (the value). Order doesn't matter here; you look up by label, not by position.

python – working with dictionaries # Create a dictionary student = { "name": "Denis", "age": 25, "course": "Python Bootcamp", "enrolled": True } # Access values by key print(student["name"]) # "Denis" print(student.get("grade", "N/A")) # safe access – returns "N/A" if not found # Add/update/delete student["grade"] = "A" # add new key student["age"] = 26 # update existing key del student["enrolled"] # delete a key # Loop through a dictionary for key, value in student.items(): print(f"{key}: {value}") # List of dictionaries – how real data looks all_students = [ {"name": "Denis", "score": 91}, {"name": "Amina", "score": 87}, {"name": "Brian", "score": 74}, ] for s in all_students: print(f"{s['name']} scored {s['score']}")
🌍 Real-World Example When you look at your Jumia order history, the backend returns a list of dictionaries. Each order is a dictionary with keys like "order_id", "item_name", "price", "status", and "date". The app loops through the list and displays each dictionary's data as a card on your screen.

🚫 Common Mistakes to Avoid
  • Index out of range: A list with 5 items has indices 0–4. Accessing index 5 crashes the program
  • KeyError with dictionaries: Accessing a key that doesn't exist crashes. Use .get("key", default) for safety
  • Confusing lists and dicts: Lists use numbers to access items (list[0]). Dicts use keys (dict["name"])

Object-Oriented Programming
Classes · Objects · Methods · Inheritance
⚡ Why This Matters As programs grow bigger, functions alone become messy. OOP lets you organise your code into self-contained objects — each combining its own data and behaviour. This is how every professional codebase is structured. Flask itself is OOP. Django is OOP. Understanding this module is the turning point from beginner to real developer.
Concept Explanation

Classes vs Objects — Blueprint vs Building

A class is like an architectural blueprint. An object is an actual building constructed from that blueprint. You define the class once, then create as many objects (instances) as you need.

python – defining a class class Student: # __init__ runs automatically when you create a new object def __init__(self, name, age, course): self.name = name # 'self' refers to the specific object being created self.age = age self.course = course self.grades = [] # starts empty for every student # A method – a function that belongs to this class def introduce(self): return f"Hi! I'm {self.name}, {self.age}, studying {self.course}." def add_grade(self, grade): self.grades.append(grade) def average_grade(self): if not self.grades: return 0 return sum(self.grades) / len(self.grades) # Create objects (instances) from the class s1 = Student("Denis", 25, "Python Bootcamp") s2 = Student("Amina", 22, "Web Design") # Use the objects print(s1.introduce()) s1.add_grade(85) s1.add_grade(92) print(f"Denis's average: {s1.average_grade()}")

Inheritance — Reusing and Extending Classes

Inheritance lets a new class inherit all the attributes and methods of an existing class, then add or override what it needs. Think of it as: a PaidStudent is a Student with extra features.

python – inheritance class PaidStudent(Student): # inherits from Student def __init__(self, name, age, course, scholarship): super().__init__(name, age, course) # run parent's __init__ self.scholarship = scholarship def introduce(self): # override the parent method base = super().introduce() return base + f" (Scholarship: {self.scholarship})" ps = PaidStudent("Brian", 20, "AI Dev", "Full Ride") print(ps.introduce()) ps.add_grade(95) # inherited from Student – works perfectly
🌍 Real-World Example In a banking app, you might have a base Account class with deposit and withdraw methods. Then SavingsAccount and LoanAccount inherit from it, each adding their own interest calculation logic. You write the shared code once and extend it — this is how Equity Bank's systems work.
⚡ Quick Challenge — Try It Now

  • Add a method age() that returns how old the car is (2026 − year)
  • Create 3 car objects and print their descriptions
🚫 Common Mistakes to Avoid
  • Forgetting self in method definitions: Every method's first parameter must be self
  • Forgetting self. when accessing attributes: Inside a method, always use self.name, not just name
  • Calling methods without parentheses: s1.introduce returns the function itself. s1.introduce() actually runs it

Python for Web Development
Flask · Routes · Templates · Running a Server
⚡ Why This Matters This is where everything clicks together. Modules 1–7 taught you Python. Now you'll use Python to build actual websites. Flask is the bridge that turns your Python code into a live web application that anyone can open in a browser. This module is the gateway to professional web development.
Concept Explanation

What Is Flask?

Flask is a micro web framework for Python. "Micro" doesn't mean limited — it means lightweight and flexible. You get the core tools you need to build web apps without a lot of unnecessary overhead. It's the most beginner-friendly way to put your Python skills on the internet.

terminal – setup # Step 1: Install Flask pip install flask # Step 2: Create your project folder mkdir my_flask_app cd my_flask_app # Step 3: Create app.py and run it python app.py # Then open: http://127.0.0.1:5000 in your browser

Routes — Mapping URLs to Python Functions

Every URL in your app maps to a Python function. The @app.route() decorator is what makes the connection. When someone visits that URL, Flask runs the function and sends back whatever it returns.

app.py – your first Flask app from flask import Flask, render_template app = Flask(__name__) # Route: / → the homepage @app.route("/") def home(): return "<h1>🚀 Welcome to VersaTech Campus!</h1><p>Built with Flask.</p>" # Route: /about → about page @app.route("/about") def about(): return "<h1>About VersaTech</h1><p>Building Africa's tech future.</p>" # Route: /greet/<name> → dynamic URL with a variable @app.route("/greet/<name>") def greet(name): return f"<h1>Hello, {name}! 👋</h1>" # Visit: /greet/Denis → Hello, Denis! if __name__ == "__main__": app.run(debug=True) # debug=True auto-reloads on save

Jinja2 Templates — Separating HTML from Python

Returning HTML strings from Python gets messy fast. Templates let you write HTML files separately and inject Python data into them using {{ }} and {% %} syntax.

templates/index.html <!DOCTYPE html> <html> <body> <h1>Welcome, {{ name }}!</h1> <ul> {# Loop through a list passed from Python #} {% for course in courses %} <li>{{ course }}</li> {% endfor %} </ul> </body> </html>
app.py – passing data to a template @app.route("/") def home(): courses = ["Python Bootcamp", "Web Design", "Data Science"] return render_template("index.html", name="Denis", courses=courses)
🌍 Real-World Example When you visit any website built with Python — like Instagram's early version, Pinterest, or Dropbox — a Flask or Django route receives your browser's request, runs a Python function, fetches data from a database, passes it to an HTML template, and sends the final page back to you. You just learned exactly how that works.
⚡ Quick Challenge — Try It Now
  • Create a Flask app with routes for /, /about, and /contact
  • Add a dynamic route /user/<username> that shows a personalised greeting
  • Create an HTML template for the homepage and pass your name and a list of hobbies to it
🚫 Common Mistakes to Avoid
  • Templates folder location: Flask looks for templates in a folder named exactly templates (lowercase) in the same directory as app.py
  • Forgetting debug=True: Without it, you have to restart the server manually every time you change your code
  • Port already in use: If you get a port error, another instance of your app is running. Close it or change the port: app.run(port=5001)

APIs
REST APIs · JSON · Flask API · requests library
⚡ Why This Matters APIs are how the modern internet works. Your weather app doesn't store weather data — it calls a weather API. WhatsApp's login connects to an authentication API. M-Pesa integrations are built on payment APIs. Learning to build and consume APIs is what separates hobby coders from professional developers.
Concept Explanation

What Is an API?

An API (Application Programming Interface) is a way for two programs to talk to each other. Your app sends a request to an API, and the API sends back a response — usually as JSON data. Think of an API like a restaurant waiter: you (the app) place an order, the waiter (API) takes it to the kitchen (server), and brings back your food (data).

JSON — The Language of APIs

JSON (JavaScript Object Notation) is the standard format for sending data between systems. It looks like Python dictionaries:

python – working with JSON import json # A JSON string (what you receive from an API) json_data = '{"name": "Denis", "city": "Narok", "score": 91}' # Parse JSON → Python dictionary data = json.loads(json_data) print(data["name"]) # "Denis" # Convert Python dict → JSON string student = {"name": "Amina", "course": "AI"} json_output = json.dumps(student, indent=2) print(json_output)

Calling External APIs with requests

terminal pip install requests
python – calling a public API import requests # Call a free public API (no key needed) response = requests.get("https://api.agify.io?name=Denis") if response.status_code == 200: # 200 means success data = response.json() # parse JSON automatically print(f"Name: {data['name']}") print(f"Predicted age: {data['age']}") else: print(f"Error: {response.status_code}")

Building Your Own REST API with Flask

api.py – full CRUD API from flask import Flask, jsonify, request app = Flask(__name__) # In-memory data store (a list of dicts) students = [ {"id": 1, "name": "Denis", "course": "Python"}, {"id": 2, "name": "Amina", "course": "Design"}, ] # GET /api/students → returns all students @app.route("/api/students", methods=["GET"]) def get_students(): return jsonify(students) # GET /api/students/1 → returns one student @app.route("/api/students/<int:student_id>", methods=["GET"]) def get_student(student_id): student = next((s for s in students if s["id"] == student_id), None) if not student: return jsonify({"error": "Student not found"}), 404 return jsonify(student) # POST /api/students → add a new student @app.route("/api/students", methods=["POST"]) def add_student(): new_student = request.get_json() new_student["id"] = len(students) + 1 students.append(new_student) return jsonify(new_student), 201 # 201 = Created if __name__ == "__main__": app.run(debug=True)
🌍 Real-World Example When a developer wants to integrate M-Pesa STK Push (the payment popup on your phone), they call the Safaricom Daraja API. They send a POST request with the phone number and amount, and the API sends back a transaction ID and triggers the payment prompt. You now have the skills to build and consume exactly this kind of system.

🚫 Common Mistakes to Avoid
  • Not checking status_code: Always check response.status_code == 200 before using the data
  • Forgetting CORS: If your frontend (HTML/JS) and backend (Flask) run on different ports, you'll need the flask-cors library
  • Wrong HTTP method: Use GET to retrieve data. Use POST to send data. Mixing them up causes errors
